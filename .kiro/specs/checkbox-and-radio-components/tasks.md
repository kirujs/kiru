# Implementation Plan: Checkbox and Radio Group Components

## Overview

This plan implements two new headless UI components (Checkbox and RadioGroup) for the headless-ui package using the Kiru framework. The implementation follows the established patterns from the Accordion component, using signal-based reactivity, context API for state sharing, and comprehensive TypeScript types. The plan includes 28 property-based tests to validate correctness properties and unit tests for specific examples and edge cases.

## Notes

- Tasks marked with `*` are optional property-based tests and can be skipped for faster MVP
- Each property test validates universal correctness properties across all inputs
- Unit tests validate specific examples, edge cases, and error conditions
- Property tests use fast-check library with minimum 100 iterations per test
- All implementation tasks reference specific requirements for traceability
- Components follow the established patterns from Accordion component
- Keyboard navigation uses createTriggerController utility
- State management uses Kiru.setup(), Kiru.derive(), and Kiru.computed()
- Ref forwarding uses createRefProxy utility
